# Design Document: PWA Install

## Overview

This feature adds a full Progressive Web App layer to the existing Next.js 16 / TypeScript / Tailwind / Supabase platform. It covers six areas:

1. **Web App Manifest** — `public/manifest.json` describing the app to browsers
2. **PNG Icon Generation** — a `scripts/generate-icons.ts` script using `sharp` to produce three icon sizes from `public/logo.svg`
3. **Service Worker** — `public/sw.js` for offline caching (cache-first static, network-first HTML) and push notification delivery
4. **InstallButton component** — a `"use client"` component embedded in the public `Header` handling `beforeinstallprompt`, `appinstalled`, standalone detection, and iOS Safari tooltip
5. **Push subscription storage** — `push_subscriptions` Supabase table, migration, RLS policies, and `savePushSubscription` server action
6. **Push notification dispatch** — `lib/push/sendPushNotification.ts` utility wired into three existing server actions in `app/actions/agent.ts`

The implementation uses no Next.js PWA plugin — the manifest is a static JSON file, the service worker is a hand-authored static JS file served from `public/`, and icon generation is a one-off build script.

## Architecture

```
Browser
  │
  ├── GET /manifest.json          ← static file, public/manifest.json
  ├── navigator.serviceWorker     ← ServiceWorkerRegistrar (client component)
  │     .register('/sw.js')
  │
  ├── InstallButton               ← "use client", mounted in Header
  │     ├── beforeinstallprompt   ← deferred prompt
  │     ├── appinstalled          ← hide button
  │     ├── standalone detection  ← window.matchMedia('(display-mode: standalone)')
  │     ├── iOS Safari detection  ← navigator.userAgent
  │     └── Enable Notifications  ← Notification.requestPermission()
  │           └── savePushSubscription (server action)
  │
  └── Push Event (sw.js)
        ├── push handler          ← showNotification(title, body, icon, data.url)
        └── notificationclick     ← clients.openWindow(data.url)

Server (Next.js server actions)
  │
  ├── savePushSubscription        ← app/actions/push.ts
  │     └── upsert push_subscriptions (Supabase admin client)
  │
  └── sendPushNotification        ← lib/push/sendPushNotification.ts
        ├── query push_subscriptions WHERE user_id = $1
        ├── webpush.sendNotification (web-push library, VAPID)
        ├── delete stale subscriptions on 410/404
        └── called from agent.ts (fire-and-forget)

Supabase
  └── push_subscriptions table
        ├── RLS: authenticated users own their rows
        └── RLS: anonymous INSERT allowed (user_id = null)

Build Script
  └── scripts/generate-icons.ts  ← sharp, run once before deploy
        ├── public/icons/icon-192.png
        ├── public/icons/icon-512.png
        └── public/icons/icon-maskable.png
```

### `public/sw.js` — Service Worker

The service worker is a plain JS file (not TypeScript, not bundled) served statically. It handles four concerns:

**Cache names** — versioned so activation can purge old caches:
```js
const CACHE_VERSION = 'v1';
const STATIC_CACHE  = `cargoplus-static-${CACHE_VERSION}`;
const HTML_CACHE    = `cargoplus-html-${CACHE_VERSION}`;
const KNOWN_CACHES  = [STATIC_CACHE, HTML_CACHE];
```

**Install handler** — pre-caches static assets:
```js
self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(STATIC_CACHE).then((cache) =>
      cache.addAll([
        '/',
        '/products',
        '/manifest.json',
        '/icons/icon-192.png',
        '/icons/icon-512.png',
        '/offline.html',
      ])
    ).then(() => self.skipWaiting())
  );
});
```

**Activate handler** — deletes stale caches:
```js
self.addEventListener('activate', (event) => {
  event.waitUntil(
    caches.keys().then((keys) =>
      Promise.all(
        keys.filter((k) => !KNOWN_CACHES.includes(k)).map((k) => caches.delete(k))
      )
    ).then(() => self.clients.claim())
  );
});
```

**Fetch handler** — cache-first for static assets, network-first for HTML navigation:
```js
self.addEventListener('fetch', (event) => {
  const { request } = event;
  const url = new URL(request.url);

  // Only handle same-origin requests
  if (url.origin !== self.location.origin) return;

  const isNavigation = request.mode === 'navigate';
  const isStatic = /\.(js|css|png|jpg|svg|ico|woff2?)$/.test(url.pathname);

  if (isNavigation) {
    // Network-first for HTML
    event.respondWith(
      fetch(request)
        .then((res) => {
          const clone = res.clone();
          caches.open(HTML_CACHE).then((c) => c.put(request, clone));
          return res;
        })
        .catch(() =>
          caches.match(request).then((cached) => cached ?? caches.match('/offline.html'))
        )
    );
  } else if (isStatic) {
    // Cache-first for static assets
    event.respondWith(
      caches.match(request).then((cached) =>
        cached ?? fetch(request).then((res) => {
          const clone = res.clone();
          caches.open(STATIC_CACHE).then((c) => c.put(request, clone));
          return res;
        })
      )
    );
  }
});
```

**Push handler**:
```js
self.addEventListener('push', (event) => {
  let payload = { title: 'CargoPlus', body: 'You have a new update', icon: '/icons/icon-192.png', data: {} };
  try {
    const parsed = event.data?.json();
    payload = { ...payload, ...parsed, icon: parsed?.icon ?? '/icons/icon-192.png' };
  } catch (_) { /* use defaults */ }

  event.waitUntil(
    self.registration.showNotification(payload.title, {
      body: payload.body,
      icon: payload.icon,
      data: payload.data,
    })
  );
});
```

**Notificationclick handler**:
```js
self.addEventListener('notificationclick', (event) => {
  event.notification.close();
  const url = event.notification.data?.url ?? '/';
  event.waitUntil(
    clients.matchAll({ type: 'window', includeUncontrolled: true }).then((windowClients) => {
      const existing = windowClients.find((c) => c.url === url && 'focus' in c);
      if (existing) return existing.focus();
      return clients.openWindow(url);
    })
  );
});
```

### `components/pwa/ServiceWorkerRegistrar.tsx`

A minimal `"use client"` component that registers the service worker on mount. Mounted once in `app/layout.tsx`.

```tsx
'use client';
import { useEffect } from 'react';

export function ServiceWorkerRegistrar() {
  useEffect(() => {
    if ('serviceWorker' in navigator) {
      navigator.serviceWorker.register('/sw.js').catch(console.error);
    }
  }, []);
  return null;
}
```

### `components/pwa/InstallButton.tsx`

A `"use client"` component. Uses a `mounted` state flag to avoid hydration mismatches — nothing renders on the server.

Key state:
- `mounted: boolean` — prevents SSR render
- `deferredPrompt: BeforeInstallPromptEvent | null`
- `isInstalled: boolean` — set on `appinstalled`
- `isStandalone: boolean` — `window.matchMedia('(display-mode: standalone)').matches`
- `isIOS: boolean` — UA detection
- `showIOSTooltip: boolean`

Visibility logic:
- Hidden if `isStandalone` (already installed)
- Hidden if `isInstalled` (just installed this session)
- Shows iOS trigger if `isIOS && !isStandalone`
- Shows standard install button if `deferredPrompt !== null`
- Shows "Enable Notifications" button always (when `mounted && !isStandalone`)

iOS detection:
```ts
const ua = navigator.userAgent;
const isIOS = /iPhone|iPad/.test(ua) && !/CriOS|FxiOS/.test(ua);
```

Notification permission flow:
```ts
async function handleEnableNotifications() {
  const permission = await Notification.requestPermission();
  if (permission !== 'granted') return;
  try {
    const reg = await navigator.serviceWorker.ready;
    const sub = await reg.pushManager.subscribe({
      userVisibleOnly: true,
      applicationServerKey: urlBase64ToUint8Array(process.env.NEXT_PUBLIC_VAPID_PUBLIC_KEY!),
    });
    await savePushSubscription(sub.toJSON());
  } catch (err) {
    console.error('Push subscription failed:', err);
  }
}
```

The `urlBase64ToUint8Array` helper converts the VAPID public key from base64url to `Uint8Array` as required by `pushManager.subscribe`.

### `scripts/generate-icons.ts`

Run once before deployment: `npx tsx scripts/generate-icons.ts`

```ts
import sharp from 'sharp';
import path from 'path';

const SVG = path.resolve('public/logo.svg');
const OUT = path.resolve('public/icons');

async function main() {
  // icon-192.png
  await sharp(SVG).resize(192, 192).png().toFile(`${OUT}/icon-192.png`);

  // icon-512.png
  await sharp(SVG).resize(512, 512).png().toFile(`${OUT}/icon-512.png`);

  // icon-maskable.png — logo at 80% (410px), centred on #4B1D8F background
  const logoSize = Math.round(512 * 0.8); // 410
  const offset   = Math.round((512 - logoSize) / 2); // 51
  const logoBuffer = await sharp(SVG).resize(logoSize, logoSize).png().toBuffer();

  await sharp({
    create: { width: 512, height: 512, channels: 4, background: '#4B1D8F' },
  })
    .composite([{ input: logoBuffer, top: offset, left: offset }])
    .png()
    .toFile(`${OUT}/icon-maskable.png`);

  console.log('Icons generated.');
}

main();
```

`sharp` is added as a dev dependency: `npm install --save-dev sharp tsx`.

## Data Models

### `push_subscriptions` Table

| Column | Type | Constraints |
|---|---|---|
| `id` | `uuid` | PK, default `gen_random_uuid()` |
| `user_id` | `uuid` | nullable, FK → `auth.users(id)` ON DELETE CASCADE |
| `endpoint` | `text` | NOT NULL, UNIQUE |
| `p256dh` | `text` | NOT NULL |
| `auth` | `text` | NOT NULL |
| `created_at` | `timestamptz` | NOT NULL, default `now()` |

Index: `idx_push_subscriptions_user_id` on `(user_id)`.

### `app/actions/push.ts` — `savePushSubscription`

```ts
'use server';
import { createAdminClient } from '@/lib/supabase/admin';
import { createServerClient } from '@/lib/supabase/server';

export async function savePushSubscription(sub: {
  endpoint: string;
  keys: { p256dh: string; auth: string };
}): Promise<{ error: string | null }> {
  try {
    const supabase = await createServerClient();
    const { data: { user } } = await supabase.auth.getUser();

    const admin = createAdminClient();
    const { error } = await admin.from('push_subscriptions').upsert(
      {
        user_id: user?.id ?? null,
        endpoint: sub.endpoint,
        p256dh: sub.keys.p256dh,
        auth: sub.keys.auth,
      },
      { onConflict: 'endpoint' }
    );
    return { error: error?.message ?? null };
  } catch {
    return { error: 'Failed to save subscription' };
  }
}
```

### `lib/push/sendPushNotification.ts`

```ts
import webpush from 'web-push';
import { createAdminClient } from '@/lib/supabase/admin';

webpush.setVapidDetails(
  'mailto:admin@cargoplus.ca',
  process.env.NEXT_PUBLIC_VAPID_PUBLIC_KEY!,
  process.env.VAPID_PRIVATE_KEY!
);

export interface PushPayload {
  title: string;
  body: string;
  data: { url: string };
}

export async function sendPushNotification(userId: string, payload: PushPayload): Promise<void> {
  const admin = createAdminClient();
  const { data: subs } = await admin
    .from('push_subscriptions')
    .select('id, endpoint, p256dh, auth')
    .eq('user_id', userId);

  if (!subs?.length) return;

  const results = await Promise.allSettled(
    subs.map((sub) =>
      webpush.sendNotification(
        { endpoint: sub.endpoint, keys: { p256dh: sub.p256dh, auth: sub.auth } },
        JSON.stringify(payload)
      )
    )
  );

  // Remove stale subscriptions (410 Gone or 404 Not Found)
  const staleIds: string[] = [];
  results.forEach((result, i) => {
    if (result.status === 'rejected') {
      const status = (result.reason as any)?.statusCode;
      if (status === 410 || status === 404) staleIds.push(subs[i].id);
    }
  });

  if (staleIds.length) {
    await admin.from('push_subscriptions').delete().in('id', staleIds);
  }
}
```

## Correctness Properties

*A property is a characteristic or behavior that should hold true across all valid executions of a system — essentially, a formal statement about what the system should do. Properties serve as the bridge between human-readable specifications and machine-verifiable correctness guarantees.*

### Property 1: Cache-first strategy for static assets

*For any* static asset URL (matching `.js`, `.css`, `.png`, `.jpg`, `.svg`, `.ico`, `.woff2`) that is present in the cache, the fetch handler SHALL return the cached response without making a network request.

**Validates: Requirements 3.3, 3.6**

### Property 2: Network-first fallback for HTML navigation

*For any* HTML navigation request (`request.mode === 'navigate'`) where the network is unavailable, the fetch handler SHALL return either the cached version of that URL or the `/offline.html` fallback — never a network error.

**Validates: Requirements 3.4, 3.6**

### Property 3: Push notification payload round-trip

*For any* valid push payload object with `title`, `body`, `icon`, and `data.url` fields, the push event handler SHALL call `showNotification` with exactly those values (using `/icons/icon-192.png` as the default icon when `icon` is absent or invalid JSON is received).

**Validates: Requirements 4.1, 4.2, 4.4**

### Property 4: Notification click opens correct URL

*For any* notification with a `data.url` value, the `notificationclick` handler SHALL call `clients.openWindow` with that exact URL (or focus an existing window at that URL).

**Validates: Requirements 4.3**

### Property 5: InstallButton visibility tracks prompt availability

*For any* sequence of `beforeinstallprompt` and `appinstalled` events, the InstallButton SHALL be visible if and only if a deferred prompt is currently held (i.e. `beforeinstallprompt` fired more recently than `appinstalled`), and SHALL remain hidden while `window.matchMedia('(display-mode: standalone)').matches` is true.

**Validates: Requirements 5.2, 5.4, 5.5, 5.6**

### Property 6: iOS Safari detection is exhaustive

*For any* user-agent string containing `iPhone` or `iPad` but not `CriOS` or `FxiOS`, the `isIOS` detection function SHALL return `true`; for all other user-agent strings it SHALL return `false`.

**Validates: Requirements 6.1**

### Property 7: savePushSubscription upsert correctness

*For any* valid `PushSubscription` JSON object, calling `savePushSubscription` SHALL result in exactly one row in `push_subscriptions` with `endpoint`, `p256dh`, and `auth` matching the input — regardless of whether the endpoint already existed (idempotent upsert).

**Validates: Requirements 7.4, 7.5**

### Property 8: RLS user isolation

*For any* two distinct authenticated users A and B, user A SHALL NOT be able to DELETE rows in `push_subscriptions` where `user_id = B.id`.

**Validates: Requirements 7.2**

### Property 9: Push dispatch covers all subscriptions

*For any* `userId` with N subscriptions in `push_subscriptions`, calling `sendPushNotification` SHALL invoke `webpush.sendNotification` exactly N times — once per subscription — and SHALL delete any subscription that returns HTTP 410 or 404.

**Validates: Requirements 9.1, 9.2, 9.3, 9.5, 9.6**

### Property 10: Notification permission gate

*For any* result of `Notification.requestPermission()` that is not `"granted"` (`"denied"` or `"default"`), the `handleEnableNotifications` function SHALL NOT call `savePushSubscription`.

**Validates: Requirements 8.2, 8.3**

## Error Handling

| Scenario | Handling |
|---|---|
| `navigator.serviceWorker.register` fails | `console.error` only — non-fatal |
| `pushManager.subscribe` throws | `console.error`, no UI error shown to visitor |
| `Notification.requestPermission` returns `"denied"` | Silently suppressed |
| `savePushSubscription` DB error | Returns `{ error: string }`, caller ignores |
| `webpush.sendNotification` returns 410/404 | Stale subscription deleted from DB |
| `webpush.sendNotification` returns other error | Logged via `Promise.allSettled` rejection, other subscriptions unaffected |
| Push payload is not valid JSON | Service worker falls back to generic notification title/body |
| `NEXT_PUBLIC_VAPID_PUBLIC_KEY` not set | `pushManager.subscribe` will throw — caught and logged |
| `VAPID_PRIVATE_KEY` not set | `webpush.sendNotification` will throw — caught by `Promise.allSettled` |
